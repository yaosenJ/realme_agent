# realme 订单价保查询技能

## 技能用途
本技能用于让智能体按照标准化流程完成 realme 订单价保查询，严格遵循固定流程执行，保证在各类场景下回复统一、流程规范、体验一致。

## 执行流程（必须严格按顺序）
1. 校验用户是否登录 realme 账号
2. 若未登录：提示未登录并结束流程
3. 若已登录：引导用户确认使用当前账号查询
4. 若用户取消或未确认：友好结束查询
5. 若用户确认：查询该账号下的 realme 订单列表
6. 若无订单：提示无订单并结束流程
7. 若有订单：引导用户选择需要查询价保的订单
8. 用户选定订单后：查询该订单的价保信息并返回价保卡片
9. 清晰展示价保金额、截止时间、价保状态等信息

## 标准回复话术
- **未登录**：
  对不起，您未登录 realme 账号，无法为您查询订单价保信息，请先登录账号。
- **确认账号引导**：
  您好，您如果咨询的是当前 realme 账号订单价保，请点击确认~
- **取消/未确认**：
  您已取消订单价保信息查询，若有其他问题可随时咨询小真喔~
- **无订单**：
  很抱歉，暂未查询到您当前 realme 账号下的订单信息。
- **选择订单引导**：
  您好，请点击您想查询价保的 realme 订单。
- **系统异常兜底**：
  很抱歉，价保查询服务暂时繁忙，请您稍后再试。

## 工具调用规则
仅按顺序使用以下 3 个工具：
1. `check_realme_login_status`：校验用户是否登录 realme 账号
2. `get_realme_user_orders`：获取当前用户账号下的 realme 订单列表
3. `get_realme_order_price_protection`：根据订单号查询对应价保信息

## 价保卡片输出格式
统一使用以下结构展示价保结果：
```
价保查询结果
订单号：{order_id}
商品名称：{product_name}
购买价格：{original_price}
当前价格：{current_price}
可价保金额：{protect_amount}
价保截止时间：{protect_deadline}
价保状态：{status}
```

## 异常与边界处理
- 订单不属于当前用户：返回权限不足，结束流程
- 订单不支持价保：明确提示该订单无可申请价保
- 价保已过期：告知用户价保已失效并展示过期时间
- 接口查询失败：使用系统异常兜底话术
