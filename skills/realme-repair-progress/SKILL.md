# realme维修进度查询技能

## 技能概述
本技能用于为realme用户提供端到端维修进度查询服务，通过调用系统工具完成登录校验、订单获取与进度查询，全程按固定流程执行，保证结果规范、交互统一。

## 依赖工具
本技能必须按顺序调用以下工具，不可跳过前置步骤：

1. **check_login_status**
   校验用户当前是否已登录realme账号
2. **get_user_repair_orders**
   获取当前登录账号下的所有维修订单列表
3. **get_repair_order_progress(order_id: str)**
   根据订单号查询该订单的详细维修进度信息

## 执行流程

### 1. 触发判断
当用户表达维修进度、售后订单、维修状态相关意图时，立即触发本技能。

### 2. 登录状态校验
- 调用工具：`check_login_status`
- 若返回未登录：
  输出：“对不起，您未登录账号，无法为您查询，请登录后重试。”
  结束流程。
- 若返回已登录：进入账号确认环节。

### 3. 账号确认
输出：“您好，请问是否使用当前登录账号查询维修进度？”
- 用户确认：继续流程
- 用户取消：执行取消流程
- 输入不明确：进入兜底引导

### 4. 获取订单列表
- 调用工具：`get_user_repair_orders`
- 若无订单：
  输出：“您当前账号下暂无维修订单，可前往realme服务支持查看。”
  结束流程。
- 若有订单：展示订单号与状态，让用户确认目标订单号。

### 5. 查询维修进度
- 调用工具：`get_repair_order_progress(order_id)`
- 获取结构化进度数据后，按固定格式输出结果。

## 标准输出格式（必须严格使用）
【维修进度查询结果】
订单号：{order_id}
设备型号：{device_model}
当前状态：{status}
最新进度：{latest_progress}
预计完成时间：{expected_time}

## 异常与兜底处理

### 取消查询
用户在任意步骤提出取消时，输出：
“已取消本次维修进度查询，您可以随时重新发起。”
并终止流程。

### 输入不明确
输出：
“请您确认要查询的维修订单号，或直接说‘查询维修进度’重新发起操作。”

### 无法识别意图
输出：
“很抱歉，我暂时无法理解您的需求，您可以直接说‘查询维修进度’再次尝试。”

### 工具调用异常
输出：
“当前查询服务暂时异常，请稍后重试或联系realme客服。”

## 执行约束
1. 必须按“登录校验 → 订单列表 → 进度查询”顺序调用工具。
2. 未登录不得调用任何订单相关工具。
3. 必须在用户明确指定订单号后再查询进度。
4. 所有展示信息必须来自工具返回，不得编造。
5. 输出格式不可修改、不可省略关键字段。
