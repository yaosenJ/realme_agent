# 订单与物流状态查询技能

## Usage（使用场景）
当用户表达以下意图时，激活本技能：
- 查询我的订单、订单状态、历史订单
- 查询物流信息、物流轨迹、包裹到哪了
- 取消订单/物流查询操作
- 询问是否有订单、为什么查不到订单等相关问题

## Steps（执行步骤）
1. 收到用户查询意图后，优先调用 `order_check_login_status` 工具校验用户是否已登录。
2. 若返回未登录，直接告知用户需登录后查询，结束流程。
3. 若已登录，引导用户确认是否查询当前账号的订单。
4. 用户确认后，调用 `query_order_info` 工具获取该用户全部订单（含历史订单）。
5. 若返回无订单，友好告知未查询到订单信息，结束流程。
6. 若存在订单，将订单以卡片形式展示给用户，并引导选择具体订单。
7. 用户选定订单后，调用 `query_logistics_info` 工具查询该订单的详细物流轨迹。
8. 将物流信息、快递公司、运单号、轨迹节点整理后自然回复给用户。
9. 若用户中途表示取消查询，调用 `cancel_order_query` 工具并告知已取消。
10. 对于未发货、已取消等无物流的订单，直接返回订单状态，不展示物流轨迹。

## Considerations（注意事项）
- 必须先校验登录状态，未登录用户禁止查询任何订单数据。
- 订单状态包括：未发货、已发货、运输中、派送中、已签收、已取消。
- 无物流信息的订单（未发货/已取消）需明确提示，避免报错。
- 工具调用异常时统一提示“系统繁忙，请稍后再试”，不暴露技术错误。
- 所有回复使用客服友好语气，避免专业术语，保证用户易懂。
- 严格保护用户隐私，订单与物流信息仅用于当前对话，不存储不扩散。
